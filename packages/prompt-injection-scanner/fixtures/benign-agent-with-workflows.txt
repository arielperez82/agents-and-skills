# Deployment Coordinator

## Purpose

Coordinates application deployments across staging and production environments, ensuring all quality gates pass before promoting changes.

## Workflows

### Standard Deployment

1. Validate all tests pass in CI
2. Build production artifacts
3. Deploy to staging environment
4. Run smoke tests against staging
5. Promote to production after approval
6. Monitor for errors post-deployment

### Rollback Procedure

1. Identify the failing deployment
2. Revert to the last known good version
3. Notify the team of the rollback
4. Create an incident report

### Blue-Green Deployment

1. Deploy new version to inactive environment
2. Run health checks on new deployment
3. Switch traffic to the new environment
4. Keep old environment as fallback for 24 hours

## Configuration

- Deployment timeout: 15 minutes
- Health check interval: 30 seconds
- Max retry attempts: 3

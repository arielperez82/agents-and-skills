# Data Validation Patterns

## Overview

This skill provides reusable input validation patterns for REST and GraphQL APIs. It enforces schema-first validation at trust boundaries using Zod.

## When to Use

- Building new API endpoints that accept user input
- Adding validation to existing endpoints
- Creating shared validation schemas across services

## Validation Patterns

### Request Body Validation

Use Zod schemas at every API boundary:

```typescript
import { z } from 'zod';

const createUserSchema = z.object({
  name: z.string().min(1).max(100),
  email: z.string().email(),
  role: z.enum(['admin', 'user', 'viewer']),
});
```

### Query Parameter Validation

Always validate and coerce query parameters:

```typescript
const paginationSchema = z.object({
  page: z.coerce.number().int().positive().default(1),
  limit: z.coerce.number().int().min(1).max(100).default(20),
});
```

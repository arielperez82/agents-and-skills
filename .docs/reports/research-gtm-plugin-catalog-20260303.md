# GTM Plugin Catalog — Research & Outlines

**Date:** 2026-03-03
**Status:** Research complete, ready for build when prioritized

## Overview

7 self-contained plugins for Claude Cowork, composed from existing agents, skills, and commands in this repo. Each plugin will be a standalone ZIP (SKILL.md + references/ + scripts/) generated by a build script that cherry-picks and composes content from source skills.

**Packaging model:** `plugins/gtm/` contains only manifests and metadata. A build script reads the manifests, pulls content from source skills, composes self-contained SKILL.md files, and outputs versioned ZIP artifacts. No source duplication in the repo.

```
plugins/
  gtm/
    README.md                      # This catalog
    build.sh                       # Composes ZIPs from manifests
    manifests/
      content-seo.yaml             # Maps source skills → plugin sections
      sales-prospecting.yaml
      sales-intelligence.yaml
      product-marketing.yaml
      demand-generation.yaml
      strategic-thinking.yaml
      gtm-doc-workshop.yaml
    dist/                          # Generated ZIPs (gitignored)
      content-seo-v1.0.0.zip
      sales-prospecting-v1.0.0.zip
      ...
```

## Plugin 7: GTM Doc Workshop

**What:** Co-authoring workflow for GTM documents — proposals, pitch decks, case studies, sales collateral, competitive briefs.
**Why:** High-frequency GTM output. The 3-stage workflow (gather → refine → test) with brainstorming loops produces polished, reader-tested docs.

| Layer | Source Assets |
|-------|--------------|
| **Skills** | doc-coauthoring, brainstorming, asking-questions |

**Key frameworks to embed:**
- 3-stage co-authoring workflow:
  1. Context gathering (info dump → clarifying questions)
  2. Refinement (brainstorm 5-20 options → curate → draft → iterate)
  3. Reader testing (fresh-eyes test for blind spots)
- Brainstorming protocol (one question at a time, multiple approaches, incremental validation)
- GTM-specific document templates:
  - Sales proposal
  - Case study
  - Competitive brief
  - Product one-pager
  - Launch announcement
  - Pitch deck narrative

## Build Process (future)

1. `build.sh` reads each manifest YAML
2. For each plugin: resolves source skill paths, extracts key frameworks/sections, composes a self-contained SKILL.md
3. Copies referenced scripts (brand_voice_analyzer.py, etc.) into plugin scripts/
4. Packages as versioned ZIP: `{plugin-name}-v{semver}.zip`
5. Output to `plugins/gtm/dist/` (gitignored)

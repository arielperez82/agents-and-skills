# Jargon Checklist

Domain-specific jargon commonly found in newsletter content. For each term, use the plain alternative or provide a brief inline explanation on first use.

**Usage:** Scan article text for these terms. If found, either replace with the plain alternative or add a brief parenthetical explanation on first use.

---

## Finance & Economics

| Jargon | Plain Alternative |
|--------|------------------|
| basis points (bps) | percentage points (1 basis point = 0.01%) |
| quantitative easing (QE) | the central bank buying bonds to inject money |
| hawkish / dovish | favoring higher rates / favoring lower rates |
| yield curve inversion | short-term rates exceeding long-term rates |
| liquidity | available cash / ease of buying and selling |
| deleveraging | paying down debt |
| headwinds / tailwinds | challenges / advantages |
| priced in | already reflected in current prices |
| risk-off / risk-on | investors avoiding risk / investors seeking risk |
| soft landing | slowing the economy without causing a recession |

## Technology

| Jargon | Plain Alternative |
|--------|------------------|
| LLM | large language model (AI that generates text) |
| fine-tuning | customizing an AI model for a specific task |
| inference | running an AI model to get results |
| latency | delay / response time |
| throughput | processing speed / volume handled |
| API | programming interface / way for software to connect |
| open source | freely available code anyone can use and modify |
| on-prem / on-premises | software running on the company's own servers |
| SaaS | software you access online (subscription) |
| moat | competitive advantage that's hard to copy |

---

## Context Rules

1. **First use matters:** If you must use a jargon term (e.g., for precision), define it on first use with a parenthetical
2. **Audience-aware:** A finance newsletter can use "basis points" without explanation; a general newsletter cannot
3. **Acronyms:** Always expand on first use unless universally known (GDP, FBI, CEO are OK)
4. **Don't over-simplify:** "The Federal Reserve" is precise, not jargon. Don't simplify to "the bank"
